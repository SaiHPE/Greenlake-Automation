# HPE Compute Ops Management Webhooks

## Webhook Definition

A webhook is an HTTP-based callback function that allows lightweight, event-driven communication between two APIs.

## Using Compute Ops Management Webhooks

Webhooks in Compute Ops Management allow for opting in to receive the latest representation of a resource whenever that resource changes state. To get started using webhooks, see the [Getting Started Guide](/docs/greenlake/services/compute-ops-mgmt/webhooks/guide). For information about the resources Compute Ops Management can send events for, and the properties that can prompt events to be sent, see [Available Resources](/docs/greenlake/services/compute-ops-mgmt/webhooks/resources).

Webhooks are highly configurable, but you must set filters when you create a webhook. To understand filter options, see the [Getting Started Guide](/docs/greenlake/services/compute-ops-mgmt/webhooks/guide) and the [Filtering](/docs/greenlake/services/compute-ops-mgmt/webhooks/filters) documentation.

Security is always a priority, and to better understand the initial registration with Compute Ops Management webhooks, review the [Webhook Prerequisites](/docs/greenlake/services/compute-ops-mgmt/webhooks/prereqs) section.
