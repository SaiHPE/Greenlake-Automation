# Reconfigure virtual machine hardware configurations

Updates CPU, memory, network adapters, and disks of a virtual machine. This operation can be performed when the virtual machine is powered off.

Endpoint: POST /virtualization/v1beta1/virtual-machines/{vm-id}/update-hardware
Version: 1.2.0
Security: bearer

## Path parameters:

  - `vm-id` (string, required)
    UUID string uniquely identifying the virtual machine
    Example: "9b4c14a6-3cd5-4907-97c4-cf44c5b641e4"

## Request fields (application/json):

  - `cpuMemConfig` (object)
    Reconfigure CPU and memory values for a virtual machine

  - `cpuMemConfig.cpu` (object)
    Reconfigure CPU values for a virtual machine

  - `cpuMemConfig.cpu.numOfCoresPerSocket` (integer)
    The number of CPU cores per socket in the virtual machine. The number of CPU cores in the virtual machine must be a multiple of the number of cores per socket.
    Example: 7

  - `cpuMemConfig.cpu.numOfCpus` (integer)
    The number of CPU cores in the virtual machine. The supported range of CPU depends on guest operating system and virtual hardware version of the virtual machine.
    Example: 35

  - `cpuMemConfig.memory` (object)
    Reconfigure memory for a virtual machine. The supported range of memory depends on guest operating system and virtual hardware version of the virtual machine.

  - `cpuMemConfig.memory.memoryInMb` (integer)
    New memory size in mebibytes.
    Example: 2048

  - `networkAdapters` (array)
    Reconfigure network adapter(s) for a virtual machine. ADD/EDIT/DELETE list of network adapters.

  - `networkAdapters.connectAtPowerOn` (boolean)
    Specifies if the network adapter has to be connected to virtual machine on power on.
    Example: "true"

  - `networkAdapters.name` (string)
    Name of the network adapter. Mandatory parameter for EDIT and DELETE operation. Name for the network adapter will be auto-generated by hypervisor-manager during ADD operation.

  - `networkAdapters.networkDetails` (object)
    Backing network information. Mandatory parameter for ADD and EDIT operation.

  - `networkAdapters.networkDetails.name` (string)
    Name of the backing network. Mandatory parameter for ADD and EDIT operation.
    Example: "VM Network"

  - `networkAdapters.networkDetails.type` (string)
    Type of the backing network.
    Enum: "STANDARD_PORT_GROUP"

  - `networkAdapters.operation` (string)
    Operation to be performed on a network adapter.
    Enum: "ADD", "EDIT", "DELETE"

  - `networkAdapters.type` (string)
    Type of the network adapter. Mandatory parameter for ADD operation.
    Enum: "E1000", "E1000E", "PCNET32", "VMXNET", "VMXNET2", "VMXNET3"

  - `virtualDisks` (array)
    Reconfigure disk(s) for a virtual machine. ADD/EDIT/DELETE list of disks.

  - `virtualDisks.diskConfig` (object)
    Configurations for a disk

  - `virtualDisks.diskConfig.capacityInMb` (integer)
    Size of the disk in mebibytes. Mandatory parameter for ADD and EDIT operation.
    Example: 2048

  - `virtualDisks.diskConfig.id` (string)
    The UUID of the disk. Mandatory parameter for EDIT and DELETE operation.
    Example: "26863c35-916e-5d00-8673-b81c48c07783"

  - `virtualDisks.diskConfig.retainFiles` (boolean)
    Specifies if the backing files of the disk need to be retained after disk deletion. Mandatory parameter for DELETE operation.
    Example: true

  - `virtualDisks.diskConfig.type` (string)
    The type of the disk.
    Enum: "IDE", "SCSI", "SATA", "NVME"

  - `virtualDisks.operation` (string)
    Operation to be performed on a disk. Minimum size required to  add a disk is 1MB. Maximum capacity for a disk depends on the backing storage.
    Enum: "ADD", "EDIT", "DELETE"
